# suona 🎺
```suona``` 微服务场景下，**一个去中心化的集群方法同步执行框架**

## 场景
比如在使用 ```caffine``` / ```hashMap``` 作为服务节点的内存缓存的时候，这是有缺点的，因为在集群内，如果需要进行缓存清理操作，是需要对每个节点进行缓存清理的，那么如果需要解决这种方案，就得通过一些中间件，如 ```MQ``` 等中间件来实现广播操作，那么如果中间件挂了，或者是不想那么麻烦依靠外部的组件来进行怎么办呢？  

那么，可以尝试使用 ```suona``` ，旨在能够让开发者在单一节点的方法被调用的时候，触发其他节点的同步进行调用，来模拟广播的操作，而开发者仅需要使用一个注解来标记这个方法即可，即可达到拆箱即用的效果
